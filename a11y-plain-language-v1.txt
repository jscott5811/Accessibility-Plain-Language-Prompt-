# Design Document: Accessibility & Plain Language

## Overview

This document defines the standards for **Plain Language** and **Easy Read** writing and formatting. The goal is to make information clear, direct, and accessible the first time it is read.

## Typography

The design system uses clean sans-serif fonts (like Open Sans) to ensure maximum readability for all users.

- **Plain Language:** Requires a minimum of 14pt text size.
- **Easy Read:** Requires a minimum of 18pt text size for users with cognitive disabilities.

### Typography (detailed rules)

- Use a clean sans-serif font such as **Open Sans**.
- Plain Language (body-md): Use 14pt text with 23pt leading.
- Easy Read (body-lg): Use 18pt text with 28pt leading.

## Layout

To prevent visual fatigue and support reading flow:
- **Alignment:** Always **left-align** text. Never justify text.
- **White Space:** Use generous spacing between paragraphs.
- **Line Breaks:** No hyphenation or word breaks across lines.
- **White space (Easy Read):** Use a maximum of 5 paragraphs per page for Easy Read.
- **Topic Breaks:** Start every new topic on a new page.
- **Safety:** Avoid runts, orphans, and widows.

## Sentence and Punctuation Rules

- **Sentence Length:** Sentences should stay between **10 and 15 words**.
- **Bullet Point Punctuation:** Bulleted items should start with a capital letter. Do not add a period at the end of bullet items unless the bullet is a full sentence.

## Do's and Don'ts

### Plain Language (6th–8th Grade Level)
- **Do** use the active voice.
- **Do** use direct address ("you" and "we").
- **Do** use common, everyday words.
- **Don't** use jargon, acronyms, or technical terms without definitions.
- **Don't** use metaphors, idioms, or sarcasm.

### Easy Read (3rd–5th Grade Level)
- **Do** limit each line to one idea.
- **Do** use concrete examples and third-person scenarios (e.g., "Jim has a job").
- **Do** bold important terms the first time they appear.
- **Don't** use abstract concepts without literal icons.

## Visual Icons

- Provide one clear, literal icon per sentence or paragraph where helpful.
- **Diversity:** Ensure icons depict diverse people, including different races and genders.
- **Literal Meaning:** Icons must be literal and avoid metaphors; they should directly represent the action or object described.

## Quick Comparison

| Feature | Plain Language (Primary) | Easy Read (Alternative) |
| :--- | :--- | :--- |
| **Reading Level** | 6th–8th Grade | 3rd–5th Grade |
| **Visuals** | Clean layout, optional images | One literal icon per sentence |
| **POV** | Second person ("You") | Third-person ("Jim") |
| **Sentence Length** | Short & direct (10–15 words) | One idea per line |
| **Font Size** | 14pt minimum (body-md: 14pt / 23pt leading) | 18pt minimum (body-lg: 18pt / 28pt leading) |
