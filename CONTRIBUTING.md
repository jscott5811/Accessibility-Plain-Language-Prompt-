# Contributing

Thank you for your interest in contributing to Accessibility-Plain-Language-Prompt-! Contributions (issues, examples, documentation, and pull requests) are welcome.

Getting started:

1. Fork the repository and create a named branch for your change: `git checkout -b feat/my-change`.
2. Make changes in a topic branch and include tests/examples where applicable (examples/).
3. Commit with clear messages and open a Pull Request against `main`.

Pull request checklist:
- [ ] Follows the repository's Plain Language and Easy Read guidelines when adding or editing content.
- [ ] Includes examples or test cases for the new content or prompt behavior.
- [ ] Documentation updated (README, docs/) when behavior or usage changes.

Review process:
- A repository maintainer will review your PR and may request changes. Please respond to feedback.

Community guidelines and reporting:
- If you see unacceptable behavior, please open an issue flagged as a Code of Conduct violation or contact the maintainers.
